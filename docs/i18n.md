# Internationalization

A cell's metadata can contain an `i18n` object. If present, the renderer will replace the contents of this object into the cell's source.

This approach is only recommended for markdown cells (i.e., changing the locale of the curriculum.) While the i18n metadata also works on code cells, it can lead to unexpected behavior (i.e., overwriting a student's code when they change language.)

## Example

In the following example, the initial default source (instructions for updating variables in English) will be replaced by the hi-IN or ja-JP equivalents, if the language is set to that locale.

```json
{
  "cell_type": "markdown",
  "metadata": {
  "i18n": {
    "hi-IN": [
    "## यह करके देखें: चर अपडेट करना\n",
    "<br>\n",
    "\n",
    "1. एक नए व्यक्ति को दर्शाने के लिए `name` और `favorite_food` के मान अपडेट करें।\n",
    "2. अपने साथी के साथ चर्चा करें: आपको क्या लगता है कि जब आप \"रन\" पर क्लिक करेंगे तो क्या होगा?\n",
    "3. प्रोग्राम चलाएं और आउटपुट देखें।"
    ],
    "ja-JP": [
    "## やってみよう: 変数の更新\n",
    "<br>\n",
    "\n",
    "1. `name` と `favorite_food` の値を新しい人物を反映するように更新してください。\n",
    "2. パートナーと話し合いましょう: 「実行」をクリックしたとき、何が起こると思いますか？\n",
    "3. プログラムを実行して、出力を観察してください。"
    ]
  }
  },
"source": [
  "## Try This: Updating Variables\n",
  "<br>\n",
  "\n",
  "1. Update the values of `name` and `favorite_food` to reflect a new person.\n",
  "2. Discuss with your partner: What do you think will happen when you click \"Run\"?\n",
  "3. Run the program and observe the output."
  ]
}
```
